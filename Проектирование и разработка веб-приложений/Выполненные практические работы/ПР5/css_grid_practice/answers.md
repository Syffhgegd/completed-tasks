# Ответы на вопросы по CSS Grid

1. **Основное свойство — `display: grid`.** Оно превращает элемент в двумерный контейнер, где дочерние элементы можно позиционировать **и по горизонтали, и по вертикали**. В отличие от Flexbox (одномерный: либо строка, либо колонка), Grid управляет **целой сеткой** — это его главное преимущество для сложных макетов.

2. **`grid-template-columns`** задаёт **явные** колонки (их количество и размеры).  
   **`grid-auto-columns`** определяет размер **неявных** колонок — тех, что создаются автоматически, когда элементы выходят за пределы явной сетки.  
   Используйте `grid-template-columns` для фиксированной структуры, а `grid-auto-columns` — при динамическом количестве элементов.

3.  
   - `repeat(3, 1fr)` → создаёт 3 колонки по 1fr.  
   - `minmax(150px, 1fr)` → колонка не уже 150px, но может расти.  
   - `auto-fill` → создаёт **максимально возможное** число колонок, даже если они пустые.  
   Пример: `grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));` — адаптивная сетка карточек.

4. **Именованные области** (`grid-template-areas`) позволяют визуально описать макет текстовой «картой», где каждая ячейка — имя области. Преимущества:  
   - Читаемость кода,  
   - Простое перемещение блоков (меняешь только имя в `grid-area`),  
   - Легкое создание адаптивных макетов (можно переопределить `grid-template-areas` в медиа-запросе).

5. **Grid-линии** нумеруются с 1 по горизонтали и вертикали (включая конечные).  
   Пример: `grid-column: 2 / 4;` — элемент занимает колонки от линии 2 до 4 (т.е. 2–3).  
   Аналогично: `grid-row: 1 / span 2;` — с 1-й строки на 2 строки вниз.

6. **`gap`** (ранее `grid-gap`) задаёт **промежутки только между элементами**, не влияя на внешние границы контейнера. В отличие от `margin`, он:  
   - Не вызывает коллапса,  
   - Не требует компенсации отрицательными margin,  
   - Упрощает адаптивную верстку.

7. **`justify-items`** выравнивает элементы **по горизонтали внутри ячейки** (начало, центр, конец).  
   **`align-items`** — **по вертикали**.  
   Оба применяются ко всем элементам сетки сразу. Для отдельного элемента используйте `justify-self` / `align-self`.

8. **`auto-fill`** создаёт столько колонок, сколько **физически помещается**, даже если они пустые.  
   **`auto-fit`** делает то же, но **схлопывает пустые колонки**, позволяя остальным занять всё пространство.  
   → `auto-fit` лучше для карточек, которые должны растягиваться.

9. С помощью **`grid-template-areas`** можно создать сложный макет, описав его текстовой схемой:
   ```css
   grid-template-areas:
     "header header"
     "nav main"
     "footer footer";
