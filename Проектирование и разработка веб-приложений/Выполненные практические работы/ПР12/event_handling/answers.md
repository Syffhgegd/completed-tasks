# Ответы на вопросы для самоконтроля (ПР12)

1. **`preventDefault()` vs `stopPropagation()`**  
   - `preventDefault()` — отменяет **стандартное действие** браузера (переход по ссылке, отправка формы).  
   - `stopPropagation()` — останавливает **всплытие события** вверх по DOM-дереву.  
   → Используйте `preventDefault` для кастомной обработки форм/ссылок; `stopPropagation` — чтобы событие не сработало на родителе (например, клик внутри модального окна не должен его закрывать).

2. **Делегирование событий**  
   Обработчик вешается на **общего родителя**, а внутри проверяется `event.target`.  
   Преимущества:  
   - Не нужно вешать обработчик на каждый элемент (экономия памяти).  
   - Автоматическая поддержка **динамически созданных** элементов.  
   Пример: `list.addEventListener('click', e => { if (e.target.matches('.delete')) … })`.

3. **Фазы события**  
   - **Capturing (захват)**: событие идёт от `window` к целевому элементу.  
   - **Target**: событие достигло цели.  
   - **Bubbling (всплытие)**: событие поднимается от цели к `window`.  
   По умолчанию обработчики работают на фазе **bubbling**. Чтобы использовать capturing — `addEventListener(..., true)`.

4. **`event.target` vs `event.currentTarget`**  
   - `target` — элемент, на котором **возникло** событие (может быть потомком).  
   - `currentTarget` — элемент, на котором **висит** обработчик (тот, у кого вызван `addEventListener`).  
   → При делегировании: `target` — кнопка ×, `currentTarget` — `<ul>`.

5. **Кастомные события**  
   ```js
   const event = new CustomEvent('myEvent', {
     detail: { data: 'payload' }, // передаём данные
     bubbles: true,    // пузырится
     cancelable: true  // можно отменить preventDefault()
   });
   element.dispatchEvent(event);
