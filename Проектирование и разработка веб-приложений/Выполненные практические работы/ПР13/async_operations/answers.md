1. **`.then/.catch` vs `async/await`**  
   `.then/.catch` — функциональный стиль, подходит для цепочек; `async/await` — императивный, читабельнее для последовательных операций.  
   Используйте `async/await` в большинстве случаев, особенно при смешанной синхронной/асинхронной логике.

2. **`Promise.all` и ошибка**  
   Если один промис отклоняется — весь `Promise.all` завершается с ошибкой **немедленно**, остальные промисы не отменяются, но их результат игнорируется.

3. **`all` vs `race` vs `allSettled`**  
   - `all` — ждёт **все**, падает при первой ошибке.  
   - `race` — ждёт **первый** завершённый (успех или ошибка).  
   - `allSettled` — ждёт **все**, возвращает массив `{status, value/reason}`.  
   → `all` — массовая загрузка, `race` — таймаут/резервный источник, `allSettled` — отчёт по всем операциям.

4. **Retry с backoff**  
   ```js
   async function retry(op, n = 3) {
     for (let i = 0; i < n; i++) {
       try { return await op(); } catch (e) {
         if (i === n-1) throw e;
         await delay(2**i * 1000);
       }
     }
   }
