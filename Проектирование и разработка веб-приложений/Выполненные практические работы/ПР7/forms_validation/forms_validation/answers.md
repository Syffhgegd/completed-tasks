# Ответы на вопросы по валидации форм

1. **HTML5-атрибуты для валидации**:  
   - `required` — поле обязательно для заполнения,  
   - `minlength` / `maxlength` — ограничение длины,  
   - `pattern` — регулярное выражение для проверки формата,  
   - `type="email"`, `type="url"` и др. — встроенная проверка типа.  
   Они обеспечивают **базовую клиентскую валидацию без JavaScript**.

2. **`novalidate`** отключает **встроенную браузерную валидацию** при отправке формы, но **не отключает API валидации** (`checkValidity()`). Это позволяет реализовать **кастомную валидацию через JavaScript**, сохраняя контроль над UX.

3. **`<label for="id">`** связывает метку с полем ввода по `id`. Это критично для:  
   - Доступности (скринридеры читают метку),  
   - Удобства (клик по метке фокусирует поле),  
   - Соответствия WCAG.

4. **Типы `input`**:  
   - `email` — для email,  
   - `tel` — для телефона (не валидирует, но вызывает цифровую клавиатуру),  
   - `password` — скрывает ввод,  
   - `date` — календарь,  
   - `checkbox` / `radio` — выбор.

5. **`pattern`** задаёт регулярное выражение. Браузер проверяет значение при отправке. Используется, когда встроенные типы недостаточны (например, сложный формат телефона или пароля).

6. **События JavaScript**:  
   - `blur` — проверка после ухода с поля,  
   - `input` — очистка ошибки при исправлении,  
   - `submit` — финальная проверка перед отправкой.  
   Это даёт **мгновенную обратную связь** без отправки формы.

7. **Доступность ошибок**:  
   - Сообщения должны быть **видимы** и **ассоциированы с полем** через `aria-describedby`,  
   - Использовать контрастные цвета + иконки,  
   - Не полагаться только на цвет (для дальтоников).

8. **`<fieldset>` и `<legend>`** группируют связанные поля (например, «Пол»). Это:  
   - Улучшает семантику,  
   - Помогает скринридерам понять структуру,  
   - Упрощает стилизацию.

9. **Кастомная валидация** (например, совпадение паролей) реализуется через JavaScript:  
   - Сравнение значений двух полей,  
   - Отображение ошибки под нужным полем,  
   - Блокировка отправки при несоответствии.

10. **Визуальные состояния**:  
    - Цвет границы (`border-color: red/green`),  
    - Фон (`background-color`),  
    - Анимации (`shake` при ошибке),  
    - Иконки или текстовые подсказки.  
    Всё это улучшает UX и снижает количество ошибок.
